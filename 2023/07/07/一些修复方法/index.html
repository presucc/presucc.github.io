pandoc exited with code null

新电脑没装 pandoc.

npm 安装时卡在 sill idealTree buildDeps

（转载自 https://blog.csdn.net/weixin_60414376/article/details/124186484）

造成上述问题的原因是 node 的默认安装环境在国外，因此我们只需要修改下镜像的地址即可.

detected dubious ownership in repository at...

新电脑用户名和旧电脑不一样，git 不信任.准确来说是仓库的文件夹所有者的名字，和现在的电脑用户名匹配不上了.

解决方法：

把不信任的那个目录全加进信任列表里.
